Learning Phases
Phase 01
Python, Math & ML Foundations
Weeks 1 – 3
1

Everything downstream depends on this. Build genuine fluency in Python and the math underlying all ML — not just enough to run a notebook, but enough to read a paper and debug a failing training run. Shortcuts here compound into confusion at every later phase.

Core Topics
🧮
Linear Algebra
Vectors, matrices, dot products, eigenvalues, SVD — the backbone of every model weight and embedding space
∂
Calculus & Optimization
Gradients, partial derivatives, chain rule, gradient descent, SGD, Adam, learning rate intuition
🎲
Probability & Statistics
Distributions, Bayes' theorem, MLE, expectation, entropy — essential for understanding model outputs and loss functions
🐍
Python for Data Science
NumPy vectorized ops, Pandas for tabular data, Matplotlib/Seaborn for EDA and visualization workflows
📊
Classical ML Algorithms
Linear/logistic regression, decision trees, SVMs, k-NN, Random Forest, XGBoost — build intuition not just API calls
🔬
Model Evaluation
k-fold CV, bias-variance tradeoff, precision/recall, ROC-AUC, F1, calibration — knowing when a model is actually good
Tools & Libraries
Python 3.12NumPyPandasScikit-learnMatplotlibSeabornJupyter / VS CodeGit & GitHub

Phase 02
Neural Networks & Deep Learning
Weeks 4 – 6
2

Implement backpropagation from scratch before touching PyTorch — this single exercise builds more intuition than weeks of reading. Then master CNNs, RNNs, and the Transformer architecture that underpins every major AI system in production today.

Core Topics
⚡
Backpropagation from Scratch
Derive and implement gradients manually — build Karpathy's micrograd, understand autograd engines
🔥
PyTorch Core
Tensors, autograd, nn.Module, custom DataLoaders, training/validation loops, checkpointing
🖼️
Convolutional Networks
Conv/pooling/BN layers, ResNet, EfficientNet — transfer learning on custom datasets end-to-end
🔄
Sequence Models
RNN, LSTM, GRU — vanishing gradients problem, gating mechanisms, when sequences matter
🎯
Transformers (Architecture)
Scaled dot-product attention, multi-head attention, positional encoding, encoder-decoder architecture
🛠️
Training Best Practices
BatchNorm, dropout, mixed precision (bf16/fp16), cosine decay + warmup, gradient clipping, weight decay
Tools & Libraries

Phase 03 New
NLP: Text to Transformers
Weeks 7 – 8
3

Before working with LLMs as black boxes, understand how language is represented computationally — from raw text to embeddings to BERT. This phase bridges classical NLP with the modern transformer-based stack and gives you the vocabulary to speak fluently about language models.

Core Topics
📝
Text Preprocessing
Tokenization (BPE, WordPiece, SentencePiece), stemming, lemmatization, stopwords, normalization
🔢
Text Representations
Bag-of-Words, TF-IDF, n-grams — understanding sparse vs dense representations and their tradeoffs
🧬
Word Embeddings
Word2Vec (skip-gram, CBOW), GloVe, FastText — how meaning is encoded in vector space geometry
🤗
BERT & Encoder Models
Masked LM pretraining, NSP, fine-tuning BERT/RoBERTa for classification, NER, QA tasks
🏷️
Core NLP Tasks
Sentiment analysis, NER, POS tagging, text classification, summarization, machine translation
📐
Sentence Embeddings
Sentence-BERT, cosine similarity, semantic search — foundational for RAG systems in Phase 4
Tools & Libraries
HuggingFace TransformersTokenizersspaCyNLTKSentence-TransformersGensimdatasets (HF)

Phase 04
LLMs, GenAI & RAG Systems
Weeks 9 – 12
4

The defining skill of 2025–2026 AI engineering. Learn to use frontier models with surgical precision, treat prompt engineering as an empirical discipline, build production RAG systems, and fine-tune open-source models efficiently on a single consumer GPU.

Core Topics
🤖
LLM API Mastery
OpenAI (GPT-4o, o3), Anthropic (Claude 3.7), Google (Gemini 2.0) — structured outputs, tool calling, streaming, batching
💬
Prompt Engineering
System prompts, chain-of-thought, few-shot, ReAct, self-consistency, XML structuring, meta-prompting
📚
RAG Pipelines
Chunking strategies (semantic, recursive), dense embeddings, BM25, hybrid search, cross-encoder reranking
🗄️
Vector Databases
Pinecone, ChromaDB, Weaviate, pgvector — ANN indexing (HNSW), metadata filtering, namespace isolation
🔧
PEFT & Fine-Tuning
LoRA, QLoRA (4-bit), PEFT library — fine-tune Llama/Mistral on a single A100/RTX 3090 with Unsloth
🧠
Reasoning Models
o1/o3, DeepSeek-R1, Qwen-QwQ — chain-of-thought at inference time, when to use thinking models vs fast models

Intern Success Principles
01
Build, Don't Just Read
Every concept should become code within 48 hours. Passive learning creates an illusion of competence that collapses under interview pressure.
02
Understand the Why
Don't just know that dropout prevents overfitting — understand why, mathematically. First-principles thinking lets you debug what tutorials don't cover.
03
Ship Weekly
A small deployed thing beats a perfect half-finished thing. Build the habit of shipping early. Momentum compounds; perfectionism stalls careers.
04
Cost & Latency Matter
In production, a model 2% less accurate but 10× cheaper and 50ms faster often wins. Think like an engineer, not a researcher chasing benchmark scores.
05
Read Original Papers
Blog posts summarize; papers contain. Read "Attention Is All You Need", LoRA, RAG, DeepSeek-R1 — primary sources build understanding blogs cannot.
06
Document Everything
Log your experiments. Write up your failures. Engineers who can explain what they tried and why it failed are far more hireable than those who can't.
